Brief language history

  • 1980 HTML (invented by Tim Bernes-Lee)
  • 1991 HTML 1.0 (first spec - HTML tags)
  • 1995 HTML 2.0 (tables, forms)
  • 1997 HTML 4.0 (CSS, frames)
  • 2000 XHTML 1.0 (XML strict syntax)
  • 2008 HTML 5 (we'll talk about it now ;-)
HTML = content to share

CSS = how it will look like

JS - smart and interactive

Elements marked as deprecated

These are: applet, center, font, frame, frameset, noframes, big, strike, u, tt, xmp, acronym, basefont, dir, isindex, noscript

Many widely used elements are to be avoided now and you should change your your way of using others as well (like embed for plugins, instead of object or mark instead of strong, other meaning of i, b, em etc.)

Deprecated attributes

for some popular elements are: link[target], html[version], img[name,hspace,vspace], object[classid,codebase...], *[align,valign], table,td,th[width,bgcolor,border,cellspacing,cellpadding], br[clear], hr[size,noshade], td,th[nowrap], ul,ol[type]   ...and there is much more of them in the specification, but I've rarely seen them to be widely adopted and in common use...

Placeholder text


  <input type=text 
    placeholder="Input hint...">

It displays descriptive text/hint in the input as long as there is no user input presented. Applicable to most of the input elements, except of radio, checkbox, range and select.

Graceful degradation = different, less comfortable or fair user experience, while still keeping the very basic purpose (like Minimum Viable Product).
Progressive enhancement = reversed strategy and generally better approach. Start with the minimal must-have and everything else is only more nice way.

Application cache for offline use

<html manifest="/offline.appcache">

File with mimetype text/cache-manifest is:

CACHE MANIFEST #modify content to re-cache
CACHE: #what to store
/css/offline.css
/js/screen.js
/img/logo.png
FALLBACK: #map online to offline URL
/ /offline.html
NETWORK: #available only when online
*

Wildcards are possible to use in FALLBACK and NETWORK sections only, not CACHE!!

HTML5 specification is in working draft, but there is no reason to hesitate - it will not be proclaimed final before at least two complete implementations!

Local and Session Storage

<script>
 localStorage.user = 'Marek Raida';
 sessionStorage.userAge = '35'; 
 alert('Your name:'+localStorage.user);
 window.addEventListener('storage', 
   function(){event in other tab}, false);
 delete localStorage.user;  // unset
</script>

localStorage is scoped to protocol, host and port, and is not shared with server (unlike cookies). Is persistent and survives browser restart (sessionStorage ends by window close)
