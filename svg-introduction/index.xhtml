Scalable Vector Graphics (SVG) overview

  • standard approved by W3 Consorcium (W3C) since 1999 (Official SVG page)
  • SMIL (Synchronized Multimedia Integration Language), W3C standard since 1998
  • SVG exists in versions 1.0 (initial), 1.1 (introduced profiles Tiny, Full and Basic) and 1.2 (extended media and multimedia support), 2.0 is under development
  • is strictly focused on 2D graphics; CSS coexists alongside with attribute formatting
  • text XML/DOM based, targeted on web use
  • Logical structure / areas / of standard:
    • Basic Shapes, Texts and Fonts
    • Gradients, Patterns, Clipping, Masking
    • Filter Effects
    • Interactivity and Linking
    • Scripting and DOM
    • Animation

Basic structure

Mime type for standalone SVG document is: image/svg+xml
Basic SVG markup structure is very easy:

<svg xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink">
  <defs>   // definitions (library section)
    <circle r="20" id="abc" fill="gray"/>   // not directly rendered element
  </defs>
  <circle r="50"/>   // directly rendered element
  <use xlink:href="#abc"/>   // element rendered by linking
</svg>
  • svg elements can be nested (svg inside svg inside svg)
  • order of elements in document is important for rendering (there is no z-index as exists in CSS)
  • defs elements can be many and anywhere
  • namespaces are important for strict implementations (like Firefox)
  • default values in place for missing attributes (like x and y coordinates, defaults to 0, black for color etc.)
  • stroke, stroke-width and fill attributes are common to majority of shape elements

Basic elements - line and rectangular

Line - simple, good old line
Attributes: x1, y1 (coordinates of starting point), x2, y2 (coordinates of ending point)

<line x1="0" y1="0" x2="100" y2="100" stroke="blue" stroke-width="3"/>

Rect - rectangular shape; can substitute special versions of circle and ellipse
Attributes: x, y, width, height, rx, ry (radius for corners, both axis, if just one is presented, it is used symmetrically)

<rect x="0" y="0" width="100" height="100" stroke="red" 
     stroke-width="2" fill="gray" rx="5"/>

Basic elements - g and image

Group - generic grouping; very important coordinate system / manipulation element; sets its own coordination subsytem
Attributes: (common formatting ones only)

<g>
 <line x1="0" y1="0" x2="10" y2="10" stroke="magenta"/>
 <line x1="0" y1="0" x2="20" y2="20" stroke="yellow"/>
</g>

Image - way how to reference and use raster or vector images
Attributes: x, y, width, height, xlink:href

<image xlink:href="country.jpg" width="160"  height="200" x="0" y="140"/>
<image xlink:href="rectangles.svg" width="100"  height="100" x="10" y="40"/>

SVG can be used also from HTML&CSS as replacement for raster images (img element, background-image, ...)

Basic elements - circle and ellipse

Circle with CSS (and transformation and interactivity). Can be substituted fully with ellipse and exists just for better semantics and simplicity. CSS :hover (mouseover) selector can be applied to SVG elements as well.
Attributes: cx, cy (x and y coordinates of center), r (radius)

<circle cx="0" cy="0" r="50" stroke="red" 
   stroke-width="5" fill="none"/>

Ellipse - oval element with two different radiuses; stroke-dasharray="space,length" in use
Attributes: cx, cy (x and y coordinates of center), rx, xy (radiuses for both axis)

<ellipse cx="0" cy="0" rx="200" ry="20" 
         stroke="maroon" stroke-width="1" fill="white"/>

Basic elements - polygon and use

Polygon with transform (polyline = polygon without fill and without auto connects)
Attributes: points (list of point coordinate points), fill-rule

<polygon points="0,0 50,100 100,0" stroke="pink" 
     stroke-width="3" fill="beige"/>

Use - any existing element with given ID can be reused hundred times
Attributes: xlink:href (anchor name of source), all other attributes are optional

<rect id="abc" x="10" y="10" width="10" height="10" fill="blue"/>
<use xlink:href="#abc" x="100" y="100"/>

Basic elements - path

Path - with transform; can replace any other shape, is the most universal one
Attributes: d (list of point coordinate points + commands) + optional pathLength. Commands are:

  • M moveto: moves the pen to a new location
  • L lineto: Draws a line from the current point to the point
  • H horizontal lineto: Draws a horizontal line from the current point to x
  • V vertical lineto: Draws a vertical line from the current point to y
  • C curveto: Draw a cubic bezier curve to the point
  • S shorthand/smooth curveto: Draw a curve to the point
  • Q Quadratic Bezier curveto: Draw a quadratic bezier between the last point and point
  • T Shorthand/smooth quadratic Bezier curveto: Draw a quadratic bezier between the last point and point
  • A elliptical arc: Draws and arc from the current point to x, y
  • z closepath: Closes the path. A line is drawn from the last point to the first
<path d="M100,10 L100,10 40,180 190,60 z" stroke="black" fill="white"/>

Basic elements - text and textpath

Text is used for basic text representation
Attributes: x, y, many other minor parameters (+ CSS styling recommended, although direct attributes exist as well)

<text x="0" y="40" stroke="red"  style="fill:blue;font-size:20px;font-weight:bold"/>

Textpath is used to place text on the curve
Attributes: xlink:href reference to path element

<path id="curve" d="M 10 100 C 200 20 250 250 450 0"/>
<textPath xlink:href="#curve">...curved text...</>

Basic elements - linear and radial gradient

Gradient - both radialGradient and linearGradient are based on offsets and stops. Gradients are used instead of color fill...
Attributes: offset, stop-color, stop-opacity (gradientUnits, spreadMethod, x1, x2, y1, y2, fx, fy...)

<linearGradient id="abc" x1="0%" y1="0%" x2="100%" y2="0%">
  <stop offset="0%" stop-color="red"/>
  <stop offset="100%" stop-color="green"/>
</linearGradient>
<circle cx="0" cy="0" r="50" fill="url(#abc)"/>

ClipPath and Mask - with animation; used for hiding part of original element
Attributes: maskUnits, maskContentUnits, ...

<clipPath id="cliptest">
  <rect width="50" height="50"/>
</clipPath>
<mask id="masktest">
  <rect width="50" height="50" fill="gray"/>
</mask>
<image xlink:href="i.jpg" width="50" height="50" clip-path="url(#cliptest)"/>
<image xlink:href="i.jpg" width="50" height="50" mask="url(#cliptest)"/>

Basic elements - filter

Filter - complex element with many subtypes (20) and options (100+)

<filter id="X" filterUnits="objectBoundingBox" x="-25%" y="-25%" 
  width="150%" height="150%">
  <feGaussianBlur result="BlurredSource" stdDeviation="0" in="SourceGraphic"/>
</filter>
<text x="10" y="10" filter="url(#X)">SVG</text>
  • filter: container for filter effects
  • feGaussianBlur: performs a Gaussian blur on the image
  • feMerge: creates image layers on top of each other (sub-element feMergeNode)
  • feMorphology: performs a "fattening" or "thinning" on a source graphic
  • feOffset: moves an image relative to its current position
  • feBlend: composites two objects together using different blending modes
  • feColorMatrix: applies a matrix transformation
  • feComponentTransfer: performs component-wise remapping of data (sub-elements feFuncA, feFuncR, feFuncG, feFuncB)
  • feDistantLight: defines a light source
  • many others: feComposite, feConvolveMatrix, feDiffuseLighting, feDisplacementMap, feFlood, feImage, fePointLight, feSpecularLighting, feSpotLight, feTile, feTurbulence...

Basic elements - transform attribute

Transform (attribute) - very powerful element applicable to all SVG elements and groups
Types:

  • translate(x [, y]): move the points in the object by (x, y)
  • scale(sx [, sy]): scale the points along the x and y axis
  • rotate(angle [, cx, cy]): rotate around the center point (cx, cy - if given)
  • skewX(angle): skew along the x-axis
  • skewY(angle): skew along the y-axis
  • matrix(a, b, c, d, e, f): transformation matrix
<g transform="scale(5) translate(15, 15) rotate(20) skewX(20) skewY(5)">
 <rect x="10" y="10" width="5" height="5" fill="firebrick"/>
 <circle r="10" fill="seagreen" stroke="blue"/>
 <rect x="5" y="5" width="12" height="2" fill="gray" stroke="silver"/>
</g>

Animation basics - Attributes

There are many common animation attributes and many specific ones. List of the most common ones are listed below (often used value is keyword indefinite):

  • attributeName: name of the target attribute to animate
  • begin: event list to begin the animation on (time value, animation event or user event + offset support)
  • dur: duration of the animation
  • end: event list to end the animation on (time value, animation event or user event + offset support)
  • restart: when the animation can be restarted (always | whenNotActive | never)
  • repeatCount: number of times to repeat the animation
  • fill: if the animation's final value sticks or not (freeze | remove)
  • calcMode: how to interpolate the animation (discrete | linear | paced | spline)
  • values: semicolon-separated list of values to set the attribute to
  • from: starting value animated from
  • to: ending value animated to
  • by: value added to the animated attribute's value by time the animation duration is completed
  • type: sets the transformation animated (translate | scale | rotate | skewX | skewY)

Animation basics - Elements

Main animation elements (non-substitutable ones)

  • animate: the most universal animate element, animation of all non-transformation attributes
  • animateTransform: second most used animation element, used for animation transformation attributes
<circle cx="20" cy="20" r="20" fill="black">
  <animate attributeName="opacity" begin="1s" dur="2s" from="1" to="0.2" repeatCount="1" id="x"/>
  <animateTransform attributeName="transform" type="scale" begin="x.end" dur="5s" 
    values="1 1; 1 0.5; 1 1" repeatCount="indefinite"/>
</circle>

Short-hand /helping/ animation elements (substitutable ones)

  • animateMotion: very specific element for animation along the path (quite hard to substitute, but possible)
  • set: short-hand version of the animate element which always has an animation duration of 0s and freezes its result (easily substitutable)
  • animateColor: designed for manipulating colors only - for semantic reasons (substitutable by animate, not recommended to use anymore)

Animation basics - Synchronized Events Based Animation Example

Basic principle of use

  • every animation has its unique identifier
  • animation can be scheduled relatively to begin, end or another event of another animation event by referencing its id
  • event offset can be used with both plus or minus effect
  • it is called SMIL (sandwich) timing model and can give you incredible flexibility (events are relative to each other and for example extending third animation's duration in chain of ten events, you do not have to update following seven animations at all...)
<g>
  <animate id="abc" attributeName="x" dur="2s" from="0" to="99" begin="3s"/>
  <animate id="def" attributeName="opacity" begin="abc.end" dur="2s" from="1" to="0.2" repeatCount="1" fill="freeze"/>
  <animateTransform attributeName="transform" type="rotate" begin="abc.begin+1s" end="def.end-1s" 
    values="0; 360" repeatCount="indefinite"/>
</g>

SVG vs SWF vs VML

  • Vector Markup Language (VML) is Microsoft's only standard, stopped active research and development in 1998; similar to SVG, but covers primarily basic drawing
  • Shockwave Flash (SWF) - FutureWare 1995, Macromedia 1996, Adobe 2005; most popular vector standard on the web. Facts:
    • binary format, no text markup representation (SVG is text markup based and can be compressed by GZIP)
    • tools are necessary to generate SWF and only the commercial ones are a good ones (SVG can be created/generated easily, although even commercial editors are not as good as for Flash)
    • is plugin, interactivity with the browser is limited (SVG is intended to be native part of browser, although plugin solutions exists as well)
    • SWF animation model is frame based (SVG animation model is time/sandwich based - SMIL)
    • its cross browser rendering differs, although marginally (SVG in different browsers is rendered accordingly to current implementation)
    • scripting is done via ActionScript, interpreter is part of the plugin (SVG uses scripting capabilities of browser itself)
    • latest trends are to avoid plugins as much possible and prefer native browser features (HTML5), even for vectors, sound and video (dusk of the plugins)

SVG vs 2D Canvas

  • many comparisons exists, like this one at Opera Developers blog or SVG Open Presentation talk
  • canvas is for programmers only
  • canvas has excellent performance, it is pixel based, but has no DOM nor event model - everything you should keep as variables in structures with references and evaluate yourself
  • resolution dependent, it is not easily scalable
  • has no animations at all and text support is very poor and not fully standardized yet
  • user interaction events and linking globally do not exist for canvas
  • no ARIA accesssibility for canvas

Is 2D Canvas wrong? No, it's geat, but has its disadvantages and you should think which technology to choose...

SVG Support

  • all common browsers have decent SVG support, including MSIE 9 released 3/2011 (even on iPhone, where's no Flash, Java or Silverlight)
  • approximately 45% of common users have browser with native SVG support (November 2010, it's more these days)
  • approximately 20% of MSIE users have installed some SVG plugin + cca 2% are using MSIE 9 already and counting (will be pushed via Live Update in 6/2011)
  • MSIE 9 is the only browser without SMIL (and FILTER) support, but they're working on it to be in the next release (but FakeSmile can help to solve lack of SMIL support on many occasions)
  • approximately 45% of MSIE users have installed Java Runtime Environment and supports applet solution (Batik, GPAC)
  • approximately 95% of MSIE users have Flash plugin installed and can use SVG Web Toolkit solution from Google
  • disregard: we can always fallback to static bitmap image for MSIE even without hint about browser upgrade

Solving troublesome MSIE via SVG Web Toolkit

  • uses Flash as renderer; via meta/get parameter (svg.render.forceflash) can be rendering forced to be driven by browser capabilities
  • requires additional three files to be loaded, src/svg.js (28kB), src/svg.htc (6kB), src/svg.swf (52kB) => 84kB in total
  • supports embedding via <object> or <embed> tags, keeping pure SVG file as external reference
  • supports inline SVG to be part of the page via <script type="image/svg+xml"> tag, which is HTML5 recommendation
  • does not support pure SVG to be external file via <iframe> tag, however it can be combined with SVG Web Toolkit as well
  • is not recommended for very complex pages, like this presentation and not everything is supported (+ minor differences to scripting as well)
  • example of use of SVGWeb in LMC - kapitol.jobs.cz
<html>
  <head>
    <script src="../../src/svg.js" data-path="../../src/"></script>
  </head>
  <body>
    <script type="image/svg+xml">
      <svg width="250" height="150">
        <rect width="36" height="36" x="30" y="30" fill="red"/>
      </svg>
    </script>
  </body>
</html>

Different ways how to use SVG on web pages

  1. <image href="x.svg" width="99" height="99" alt="x"/>     // does not allow interactivity and scripting
  2. <div style="background-image(x.svg);width:99;height:99"/>     // does not allow interactivity and scripting
  3. <iframe src="x.svg" width="99" height="99"/>     // very simple and safe way
  4. <object data="x.svg" type="image/svg+xml" width="99" height="99"/>     // allows perfect fallback way
  5. <embed src="x.svg" type="image/svg+xml" width="99" height="99"/>     // deprecated, but allows good fallback way as well
  6. <svg xmlns="http://www.w3.org/2000/svg" width="99" height="99"/>     // direct and easy way for xHTML documents
  7. <script type="image/svg+xml"><svg width="99" height="99"/></script>     // HTML5 way with nice fallback
  8. <script type="text/javascript"><window.createElement("svg")..appendChild(e)..</script>     // via scripting and DOM
  9. <script type="text/javascript"><new XSLTProcessor()...</script>     // via scripting XSLT and innerHTML
 10. <image href="data:image/svg+xml,<svg xmlns='http://www.w3.org/2000/svg'>...</svg>" width="99" height="99" alt="x"/>   // inline image
 11. + direct show of SVG / SVGZ document, of course, base64 encoded data and couple of other possibilities... ;-)
